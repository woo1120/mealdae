# Strict Execution Rule
- **NEVER** write code, modify files, or perform any implementation work unless the user EXPLICITLY commands you to do so (e.g., "작업해", "만들어줘", "진행해", "부탁해").
- If the user asks a theoretical question or asks "Is this possible?" and appends "작업하지마" (Do not work), you must ONLY provide a text explanation and MUST NOT make any changes to the codebase or artifacts.
- Even if an implementation plan is approved, if the user says "작업하지마" or similar, halt execution.
- Always wait for clear, explicit authorization before initiating EXECUTIOn tasks.
