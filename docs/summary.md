# 내 식비 지키미 (Meal Planner) - 프로젝트 요약서

## 1. 프로젝트 개요
**내 식비 지키미(Meal Planner)**는 직장인의 월간 식비(구내식당, 외부 식사) 지출 내역을 기록하고, 예산 대비 소진율을 확인하며, 월말에 본부장님(관리자)에게 손쉽게 비용 청구(Reimbursement) 이메일을 보낼 수 있도록 돕는 단일 페이지 웹 애플리케이션(SPA)입니다.

## 2. 주요 기능 (Core Features)
* **대시보드 (Dashboard)**
  * 이번 달 총 사용한 식비, 남은 예산(기본 예산: 200,000원), 청구할 총액(페이백 대상) 표시.
  * 직관적인 예산 소진율 프로그레스 바(Budget Gauge) 제공.
* **캘린더 뷰 (Calendar View)**
  * 달력 형태의 UI를 통해 매일매일의 식사 형태(구내식당, 외식, 휴일/연차)를 기록.
  * 구내식당 비용은 상수로 고정(7,770원)되며, 외식은 식당 이름, 결제 카드, 금액, 시간(점심/저녁)을 개별 입력.
* **비용 청구(Reimbursement) 자동화**
  * 한 달 동안 기록된 '외식(Outing)' 내역과 '구내식당(Cafeteria)' 횟수를 취합하여 메일용 요약 텍스트 및 영수증 표를 자동 생성.
  * **크로스 플랫폼 지원(v1.10)**: 
    * 아이폰/PC, 안드로이드 상관없이 모든 기기에서 클립보드 복사 시 예쁜 HTML 표 양식이 메일 앱에 보존되도록 "Hidden Element Selection" 방식의 복사 로직 적용.
    * 아이폰의 다크 테마 복사 버그 방지를 위해 흰 바탕/검은 글씨를 강제 적용.
* **데이터 관리 및 백업 (Data Management)**
  * 입력된 카드사와 식당 이름은 로컬 히스토리에 자동 저장되어 다음 입력 시 자동완성(Dropdown) 제공.
  * JSON 파일 형태로 전체 데이터를 내보내기(Export) 및 가져오기(Import) 가능.

## 3. 기술 스택 및 아키텍처
본 프로젝트는 별도의 번들러나 복잡한 프레임워크 없이 순수 프론트엔드 기술만으로 가볍고 빠르게 동작하도록 설계되었습니다.

* **HTML (`index.html`)**: 메인 화면, 대시보드, 캘린더, 모달(로그인, 식사 선택, 기록 관리, 통계, 메일 전송) 영역이 구조화된 단일 문서.
* **CSS (`style.css`)**: 
  * Glassmorphism (반투명 유리) 디자인 시스템 적용 (`.glass-card`).
  * CSS 변수(`--primary`, `--accent` 등)를 활용한 일관된 테마 구성 및 다크 모드 기반의 프리미엄 UI.
* **JavaScript (`app.js`)**: 
  * 전체 상태 관리(`state` 객체).
  * DOM 조작 및 이벤트 리스닝.
  * 로컬 스토리지(`localStorage`) 기반의 데이터 영구 저장 및 API 연동(Fetch 통신 준비).
  * 클립보드 API(`document.execCommand`, `navigator.clipboard.write`)를 활용한 메일 전송 로직 제어.

## 4. 데이터 플로우 (Data Flow)
1. **로그인**: 사용자가 아이디를 입력하면 `localStorage`에 아이디를 저장하고 앱을 렌더링.
2. **조회 및 초기화**: `loadUserData()` 호출 시 `localStorage` (또는 `/api/data`)에서 해당 유저의 식비 데이터를 불러옴. 해당 달의 데이터가 없으면 주말은 '휴일', 평일은 '구내식당'으로 디폴트 초기화(`initializeMonthData`).
3. **업데이트**: 캘린더에서 날짜 클릭 -> 식사 정보 모달 입력 -> `saveMeal()` 호출 시 `state.mealData` 객체 업데이트 후 `saveUserData()`로 로컬 스토리지 및 네트워크에 데이터 동기화. 화면 즉시 리렌더링.
